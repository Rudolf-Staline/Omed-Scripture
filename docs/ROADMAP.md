# Roadmap

## V0.1 beta
- Stabilize core routes, docs, CI, feedback, privacy/licensing notes and QA checklist.
- Validate Reader, Search, local stores, PWA/offline and optional Google Drive sync with beta testers.
- Keep data collection opt-in and avoid backend requirements.

## V0.2
- Improve offline clarity, search performance and static pack validation.
- Add deeper accessibility checks and keyboard QA.
- Reduce bundle size with route-level code splitting if measurements justify it.

## V0.3
- Harden sync conflict handling, restore flows and recovery messaging.
- Expand automated route smoke tests and component tests for critical pages.
- Improve mobile PWA install guidance and beta feedback triage.

## V1 stable
- Document supported Bible data sources and licensing policy completely.
- Finish accessibility/performance budgets.
- Confirm backup/sync reliability with real-world testers.
- Publish a stable release process and support expectations.
